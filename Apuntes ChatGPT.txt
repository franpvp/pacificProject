To translate your Django project from Spanish to English, you can use Django's built-in internationalization (i18n) and localization (l10n) features. Here's a step-by-step guide on how to achieve this:

Set up your project for internationalization:
First, make sure your Django project is configured for internationalization. In your settings.py file, ensure the following settings are configured:

python
Copy code
LANGUAGE_CODE = 'es'

# Add 'django.middleware.locale.LocaleMiddleware' to MIDDLEWARE
MIDDLEWARE = [
    ...
    'django.middleware.locale.LocaleMiddleware',
]

# Specify the languages your website supports
LANGUAGES = [
    ('es', 'Spanish'),
    ('en', 'English'),
]
Mark translatable strings:
In your HTML templates and Python code, wrap the text you want to translate with Django's translation tags {% trans %} and {{ _('text') }}. For example:

html
Copy code
{% load i18n %}
<h1>{% trans "Welcome" %}</h1>
In Python code:

python
Copy code
from django.utils.translation import gettext as _
print(_("Hello"))
Extract translatable strings:
Django provides a management command to extract translatable strings from your code. Run the following command in your terminal:

Copy code
python manage.py makemessages -l en
This will create .po files for English translation in your project's locale directory.

Translate strings:
Open the generated .po files in a text editor. You'll see entries for each translatable string. Translate the strings from Spanish to English.

Compile messages:
After you've translated the strings, compile the translations into .mo files by running:

Copy code
python manage.py compilemessages
Set language preference:
You can set the language preference in your Django application settings or let users choose their preferred language through a language selector. To set the language preference, you can use Django's session or cookie-based language switching mechanism.

Test your translations:
Run your Django application and test the translations to ensure they are working as expected.

By following these steps, you can translate your Django project from Spanish to English. Remember to maintain your translations as your project evolves and new strings are added.

const switchButton = document.querySelector(".switch");

document.addEventListener("DOMContentLoaded", () => {
    // Check if dark mode is saved in local storage
    cargarDarkModeDesdeLocalStorage();
    switchButton.addEventListener("click", toggleDarkMode);
});

function toggleDarkMode() {
    const isDarkMode = !switchButton.classList.contains("active");
    switchButton.classList.toggle("active");
    document.body.classList.toggle("active");

    // Add transition class to body for smooth transition
    document.body.classList.add("mode-transition");

    // Save dark mode state to local storage
    guardarDarkModeEnLocalStorage(isDarkMode);

    // Remove the transition class after a short delay
    setTimeout(() => {
        document.body.classList.remove("mode-transition");
    }, 300); // Adjust the duration to match your CSS transition duration
}

function guardarDarkModeEnLocalStorage(estado) {
    localStorage.setItem("darkMode", estado);
}

function cargarDarkModeDesdeLocalStorage() {
    // Check local storage for dark mode state
    const darkModeGuardado = localStorage.getItem("darkMode") === "true";
    if (darkModeGuardado) {
        // Apply dark mode styles if it's enabled
        switchButton.classList.add("active");
        document.body.classList.add("active");
    }
}
